# 搜索

GuideNH 会为指南搜索维护一个内存中的 Lucene 索引。

## 会索引什么

每个页面会贡献以下信息：

- 标题
- 完整可搜索正文
- guide id
- page id
- 语言元数据

### 标题来源

GuideNH 会按以下顺序选择标题：

1. `frontmatter.navigation.title`
2. 第一个 `# Heading`
3. 页面资源 id

### 正文来源

可搜索正文由解析后的 Markdown 内容，以及显式向索引贡献文本的自定义标签共同组成。

示例：

- 普通 Markdown 段落会被索引
- 标题会被索引
- tooltip 标签和链接标题可能被索引
- 命令链接标题会被索引
- 搜索不仅能匹配标题，也能匹配正文

## 结果布局

当前游戏内搜索页会显示：

- 页面图标（若存在）
- 蓝色显示的页面标题
- 右对齐的页面资源路径
- 正文中带高亮的文本片段

当查询为空时，页面会显示居中的提示信息；当没有匹配项时，会显示居中的“无结果”信息。

## 语言行为

搜索分析器会尽量根据当前 Minecraft 语言选择。未知语言会回退到英文分词。

## 查询范围

默认情况下，指南搜索最多返回 25 条结果。必要时，搜索页也可以在内部限制为单一 guide。

## 更利于搜索的编写建议

- 将真正面向人的页面名称放进 `navigation.title`。
- 重要页面最好从有意义的 `#` 标题开始。
- 重要名词应写进正文，而不仅仅放进图片或图标里。
- 如果希望用户容易搜到页面，优先在文本里写出稳定的物品名称和机器名称。

## 示例搜索目标

内置示例指南很适合作为搜索冒烟测试，因为它包含：

- Markdown 语法示例
- 配方页面
- 结构预览
- 图片资源
- 3D 场景示例

准确的运行时文件列表可参考 [示例](Examples-zh-CN)。
